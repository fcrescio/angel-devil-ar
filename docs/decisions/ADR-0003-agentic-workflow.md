# ADR-0003: Agentic Workflow

## Status

Accepted.

## Decision

The repository uses explicit local docs, small backlog tasks, done criteria, CI, issue templates, and handoff notes to support agentic development.

## Context

Agents work best when scope, authority, and task boundaries are local to the repository.

## Consequences

- `AGENTS.md` is required and must stay current.
- Meaningful architecture changes require doc updates and often an ADR.
- Work should map to small backlog entries.
- PRs must include validation and architecture impact.
